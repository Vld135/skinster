# Контекст аккаунтов

Нода **Account Context** определяет набор аккаунтов, на которых будут выполняться все последующие действия в workflow. Это обязательная нода — без неё остальные действия не будут знать с какими аккаунтами работать.

## Как это работает

1. Account Context загружает список аккаунтов из выбранного источника
2. Аккаунты "фиксируются" на всё время выполнения workflow
3. Каждая последующая нода получает доступ к этим аккаунтам
4. Ноды действий могут дополнительно фильтровать аккаунты через параметр **Аккаунты** (`contextFilter`)

## Параметры

| Параметр | Описание | Тип | По умолчанию |
|----------|----------|-----|--------------|
| Источник аккаунтов | Откуда брать аккаунты для workflow | `select` | Все аккаунты |
| Папка | Выберите папку с аккаунтами | `folderSelector` | — |
| Выбранные аккаунты | Выберите конкретные аккаунты | `accountCheckboxList` | — |

## Режимы выбора

### Все аккаунты

Загружает все аккаунты из базы данных. Подходит для операций, которые нужно выполнить на всех аккаунтах сразу.

### Папка

Загружает только аккаунты из указанной папки. Удобно если аккаунты сгруппированы по назначению (например, "Фарм", "Трейд", "Основные").

Параметр **Папка** появляется только при выборе этого режима.

### Выбранные вручную

Позволяет выбрать конкретные аккаунты из списка с чекбоксами.

Параметр **Выбранные аккаунты** появляется только при выборе этого режима.

## Выходы

| Выход | Когда |
|-------|-------|
| **success** | Аккаунты успешно загружены (хотя бы один) |
| **empty** | Список аккаунтов пуст |

## Фильтрация в действиях

Большинство нод действий имеют параметр **Аккаунты** (`contextFilter`) с двумя вариантами:

* **Все** — использовать все аккаунты из контекста
* **Выбранные** — использовать только определённые аккаунты из контекста

Это позволяет одному workflow работать с разными подмножествами аккаунтов на разных этапах.
